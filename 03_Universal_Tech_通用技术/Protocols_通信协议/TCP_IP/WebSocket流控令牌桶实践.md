# WebSocket流控令牌桶实践

**Tags:** #WebSocket #流控 #令牌桶 #音频流
**来源:** `note_of_dev/NOTE.md`（已归档）
**关联:** [[esp_websocket_client并发风险与规避]]

## 背景

ESP32-S3 作为 WebSocket 客户端进行流式音频传输。网络有抖动时，若服务端按固定速率推流，客户端缓冲区会被打满，导致丢包和播放卡顿。

## 方案

客户端根据自身消费速度向服务端下发令牌；服务端仅在令牌允许的额度内发送音频包。

## 要点

- 令牌发放周期和单位包大小要与客户端解码/播放节奏一致。
- 令牌不足时服务端必须停发，避免瞬时拥塞。
- 客户端重连后需要重置令牌状态，避免历史计数污染。
